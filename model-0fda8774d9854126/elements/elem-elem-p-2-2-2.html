Model development and semantic validation ()
The trigger for the model development and semantic validation process occurs when a scoped increment definition is received from capability increment scoping. This process encompasses the development of model changes, the execution of semantic validation checks, and the submission of validated changes for review and approval, involving stakeholders such as EA practitioners and architecture leads throughout its lifecycle. It exists to ensure that model changes are accurately developed and validated, delivering high-quality architecture documentation; without this process, the integrity of the architecture could degrade, leading to potential misalignment in the release cycle. The primary output is validated model changes submitted to review and approval coordination. <!-- SPEC --> **Lifecycle phases** - receive scoped increment definition: the process begins with the receipt of a scoped increment definition, which outlines the necessary changes to the architecture model. - develop model changes: based on the scoped increment definition, model changes are developed in accordance with the established meta-model specification and modelling conventions document. - run semantic validation checks: the developed model changes undergo semantic validation checks to ensure compliance with the defined standards and quality expectations. - resolve validation findings: any issues identified during the validation checks are addressed and resolved to maintain the integrity of the model. - submit validated model changes for review: once validation findings are resolved, the validated model changes are submitted for review and approval coordination. **Information objects** - architecture model: produced by upstream source, arrives at develop model changes, passed to run semantic validation checks - meta-model specification: produced by upstream source, arrives at develop model changes, passed to run semantic validation checks - modelling conventions document: produced by upstream source, arrives at develop model changes, passed to run semantic validation checks - quality report: produced by upstream source, arrives at run semantic validation checks, passed to resolve validation findings - validated model changes: produced by upstream source, arrives at submit validated model changes for review, passed to review and approval coordination
